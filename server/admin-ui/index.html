    Stellschrauben

    Direkt auf der Produktions-API — nur mit Absicht bedienen.

    • ADMIN_API_KEY in der Server-.env schützt alle /api/v1/admin/*-Routen.
    • Pro-Flag hier ist eine manuelle Override-Hilfe; Live-Entitlements können zusätzlich über RevenueCat laufen.
    • Kosten-Schätzung im KI-Tab: optional OPENAI_PRICE_INPUT_PER_1M_USD / OPENAI_PRICE_OUTPUT_PER_1M_USD.

    Kein klassisches „Login“. Die Admin-Routen unter /api/v1/admin/ sind mit einem gemeinsamen Geheimnis geschützt, das in der Server-.env als ADMIN_API_KEY steht. Der Browser sendet bei jedem Klick auf „Laden“ den Schlüssel als Header X-Admin-Key (alternativ Authorization: Bearer … — siehe Middleware im Repo). Speichern (lokal) legt URL und Schlüssel nur in localStorage auf deinem Rechner ab — damit du sie nicht jedes Mal neu eingeben musst. Das ist dasselbe Prinzip wie ein API-Token in Postman, nur ohne OAuth/Benutzerkonten-App für dieses interne MVP.
