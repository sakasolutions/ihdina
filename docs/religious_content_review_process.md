# Religiöse Lerninhalte — Prüf- und Freigabeprozess

Dieses Dokument beschreibt den **verbindlichen Prozess** für religiöse Lerninhalte in Ihdina.

**Geltungsbereich (Phase 1):**

- Wudu
- Ghusl
- Tayammum
- Gebet

Der Prozess kann später auf weitere Lernbereiche erweitert werden.

---

## 1. Positionierung von Ihdina

- Ihdina ist eine **Lern- und Verständnishilfe**.
- Ihdina ist **keine Fatwa-App**.
- Ihdina **ersetzt keine Gelehrten**.
- Ihdina **beantwortet keine religiösen Einzelfallfragen**.
- Die App **leitet keine eigenen Rechtsurteile** ab.
- Die **KI ist keine religiöse Autorität**.
- Lerninhalte werden **nicht ungeprüft durch KI erzeugt**.

---

## 2. Fachliche Ausrichtung

- Der erste Lernpfad folgt **transparent der hanafitischen Lehrtradition**.
- Andere Lehrtraditionen werden **nicht als falsch** dargestellt.
- Unterschiede zwischen **Fard, Wajib, Sunnah und Mustahabb** müssen klar getrennt werden.
- **Mindestanforderung** und **empfohlene vollständige Ausführung** dürfen nicht vermischt werden.
- Unklare oder strittige Punkte werden **nicht vereinfacht dargestellt**, ohne sie vorher zu prüfen.

---

## 3. Quellenhierarchie

Die Inhalte sollen möglichst auf mehreren Ebenen beruhen:

1. Koran
2. authentische Hadithüberlieferungen
3. anerkannte klassische hanafitische Lehrwerke
4. moderne institutionelle Kontrollquellen, zum Beispiel Ilmihal-Werke
5. fachliche Prüfung der fertigen Inhalte

### Bereits vorhandene Koranübersetzungen in Ihdina

- Bubenheim
- Elyas

### Als mögliche hanafitische Haupt- und Kontrollwerke bisher betrachtet

> **Offen:** Diese Liste ist noch **nicht als endgültig freigegeben** zu behandeln. Sie dient der internen Orientierung und muss vor produktiver Nutzung fachlich und rechtlich bestätigt werden.

- Nur al-Idah
- Maraqil Falah
- Mukhtasar al-Quduri
- institutionelle Ilmihal-Werke, zum Beispiel Diyanet

---

## 4. Vorgehen pro Lernschritt

Für **jeden Lernschritt** wird intern dokumentiert:

| Feld | Beschreibung |
|------|--------------|
| Eindeutige ID | Stabile Kennung im Datenmodell |
| Bereich | z. B. Wudu, Ghusl, Tayammum, Gebet |
| Titel | Anzeigename in der App |
| Lernziel | Was der Nutzer nach dem Schritt verstehen soll |
| Kurze App-Erklärung | Verständlicher Text für die App-Oberfläche |
| Einordnung | Fard, Wajib, Sunnah oder Mustahabb |
| Mindestanforderung | Was für die Gültigkeit erforderlich ist |
| Empfohlene Ausführung | Vollständige oder empfohlene Ausführung |
| Verwendete Quellen | Welche Werke/Überlieferungen herangezogen wurden |
| Genaue Fundstelle | Werk, Kapitel, Seite oder Hadithnummer |
| Offene fachliche Fragen | Noch ungeklärte Punkte |
| Prüfstatus | Aktueller Stand der fachlichen Freigabe |

### Mögliche Prüfstatus

| Status | Bedeutung |
|--------|-----------|
| Entwurf | Inhalt in Arbeit, noch nicht quellenbasiert abgeschlossen |
| Quellen geprüft | Fundstellen und Quellenbezug sind dokumentiert |
| Fachlich offen | Quellen vorhanden, fachliche Klärung ausstehend |
| Fachlich geprüft | Inhalt von qualifizierter Person geprüft |
| Zur Veröffentlichung freigegeben | Freigabe für die App-Veröffentlichung erteilt |

---

## 5. Textentstehung

- Geschützte Werke dürfen als **Fachquellen** verwendet werden.
- **Geschützter Wortlaut wird nicht kopiert.**
- App-Texte werden **vollständig eigenständig** und verständlich auf Deutsch formuliert.
- Quellenangaben **ersetzen keine Nutzungserlaubnis** für kopierten Text.
- Direkte Zitate werden nur verwendet, wenn sie **rechtlich zulässig und notwendig** sind.
- Arabische Originaltexte oder Übersetzungen werden nur übernommen, wenn die **Nutzung geklärt** ist.

---

## 6. Fachliche Prüfung

Vor Veröffentlichung soll der **vollständige Lernpfad** geprüft werden durch:

- eine Person mit **nachweisbarer hanafitischer Fiqh-Ausbildung**, oder
- eine **geeignete islamisch-theologische bzw. religionspädagogische Fachperson**

Die Prüfung soll **dokumentiert** erfolgen.

### Prüfübersicht (geplant)

> **Offen:** Eine strukturierte Prüfübersicht ist **noch anzulegen**. Geplante Spalten:

| Spalte | Zweck |
|--------|-------|
| App-Text | Der zu prüfende Inhalt |
| Quellen | Herangezogene Werke und Fundstellen |
| Einordnung | Fard / Wajib / Sunnah / Mustahabb |
| Offene Frage | Ungeklärte fachliche Punkte |
| Freigegeben | Ja / Nein |
| Korrektur erforderlich | Ja / Nein |
| Anmerkung | Zusätzliche Hinweise der Prüfperson |

### Keine unbelegten Qualitätsaussagen

Es darf **nicht** behauptet werden:

> „Von Gelehrten geprüft“

solange **keine tatsächliche, dokumentierte Prüfung** erfolgt ist.

---

## 7. Darstellung in der App

### Auf Bereichsebene

- Aufklappbare Card: **„Quellen & fachliche Grundlage“**

### Auf Ebene eines Lernschritts

- Konkrete Quellen zu **diesem Schritt**
- Angabe: Werk, Kapitel, Seite oder Hadithreferenz

### Zusätzlicher Hinweis (App-weit bzw. kontextuell)

- Lernhilfe
- Keine individuelle Fiqh-Beratung
- Bei persönlichen oder unklaren Fragen: **qualifizierte religiöse Anlaufstelle** kontaktieren

> **Offen:** Konkrete UI-Umsetzung, Texte und Platzierung in den jeweiligen Screens sind **noch zu spezifizieren** und folgen diesem Prozessdokument.
